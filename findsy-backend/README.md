# findsy backendA real backend for your findsy storefront: SQLite database, JWT-protected adminroutes, and image upload/optimization — replacing the browser `window.storage`calls and hardcoded passcode in the original HTML.## What this replaces|Old (in findsy.html)|New||-|-||`window.storage.get/set` (per-browser, fake)|SQLite database, shared for all visitors||`ADMIN\\\\\\\\\\\\\\\_PASSCODE = 'findsy2026'` in plain JS|bcrypt-hashed password + JWT login endpoint||Base64 images stuffed into localStorage|Real files on disk, resized/optimized, served via `/uploads/...`|## 1\. Install```bashnpm install```## 2\. Configure```bashcp .env.example .env```Then edit `.env`:* `CORS\\\\\\\\\\\\\\\_ORIGIN` — the URL(s) your frontend will be served from (comma-separated).* `JWT\\\\\\\\\\\\\\\_SECRET` — set a long random string, e.g. `node -e "console.log(require('crypto').randomBytes(48).toString('hex'))"`.* `ADMIN\\\\\\\\\\\\\\\_USERNAME` — whatever you want, e.g. `admin`.* `ADMIN\\\\\\\\\\\\\\\_PASSWORD\\\\\\\\\\\\\\\_HASH` — generate with:```bash  node -e "console.log(require('bcryptjs').hashSync('YOUR-REAL-PASSCODE', 10))"  ```Paste the output (starts with `$2a$...`) into `.env`. Never store the plaintext passcode.## 3\. (Optional) seed demo data```bashnpm run seed```## 4\. Run```bashnpm start```Server runs on `http://localhost:4000` by default. Health check: `GET /api/health`.## API Reference### Auth|Method|Route|Auth|Body||-|-|-|-||POST|`/api/auth/login`|none|`{ username, password }` → `{ token }`||GET|`/api/auth/verify`|Bearer token|checks if token is still valid|### Products|Method|Route|Auth|Notes||-|-|-|-||GET|`/api/products`|none|`?category=\\\\\\\\\\\\\\\&search=\\\\\\\\\\\\\\\&limit=\\\\\\\\\\\\\\\&offset=`||GET|`/api/products/categories`|none|distinct category list||GET|`/api/products/:id`|none|single product||POST|`/api/products`|admin|create — `{ title, image, link, price, oldPrice, category, desc }`||PUT|`/api/products/:id`|admin|update (partial allowed)||DELETE|`/api/products/:id`|admin|delete|### Upload|Method|Route|Auth|Notes||-|-|-|-||POST|`/api/upload`|admin|`multipart/form-data`, field name `image`. Returns `{ url }` — use that URL as the product's `image` field.|Admin routes require header: `Authorization: Bearer <token>` (token from `/api/auth/login`).## 5\. Wire up findsy.htmlIn the frontend, replace the storage/auth logic with `fetch` calls to this API. Key swaps:```jsconst API\\\\\\\\\\\\\\\_BASE = 'http://localhost:4000/api'; // or your deployed URLlet authToken = null; // store in memory or sessionStorage after login// Load products (public)async function loadProducts(){  const res = await fetch(`${API\\\\\\\\\\\\\\\_BASE}/products`);  const data = await res.json();  PRODUCTS = data.products;  renderShop();  renderAdminList();}// Admin login (replaces ADMIN\\\\\\\\\\\\\\\_PASSCODE check)async function tryUnlock(){  const password = document.getElementById('passInput').value;  const res = await fetch(`${API\\\\\\\\\\\\\\\_BASE}/auth/login`, {    method: 'POST',    headers: {'Content-Type':'application/json'},    body: JSON.stringify({ username: 'admin', password })  });  if(res.ok){    const data = await res.json();    authToken = data.token;    sessionStorage.setItem('findsy\\\\\\\\\\\\\\\_token', authToken); // survives refresh, cleared on tab close    // show dashboard...  } else {    // show "Incorrect passcode" error  }}// Create/update product (admin)async function saveProduct(payload, editingId){  const url = editingId ? `${API\\\\\\\\\\\\\\\_BASE}/products/${editingId}` : `${API\\\\\\\\\\\\\\\_BASE}/products`;  const method = editingId ? 'PUT' : 'POST';  const res = await fetch(url, {    method,    headers: {      'Content-Type': 'application/json',      'Authorization': `Bearer ${authToken}`    },    body: JSON.stringify(payload)  });  return res.json();}// Delete product (admin)async function deleteProduct(id){  await fetch(`${API\\\\\\\\\\\\\\\_BASE}/products/${id}`, {    method: 'DELETE',    headers: { 'Authorization': `Bearer ${authToken}` }  });}// Image upload (admin) — instead of canvas.toDataURL base64async function uploadImage(file){  const form = new FormData();  form.append('image', file);  const res = await fetch(`${API\\\\\\\\\\\\\\\_BASE}/upload`, {    method: 'POST',    headers: { 'Authorization': `Bearer ${authToken}` },    body: form  });  const data = await res.json();  return data.url; // use this as the product's `image` field}```I can make these exact edits directly in your `findsy.html` if you'd like — just say so.## 6\. DeployingThis is a standard Node/Express app, so it runs on Render, Railway, Fly.io, a VPS,etc. Notes:* `better-sqlite3` is a native module — most PaaS providers build it fine ondeploy, but if your host is read-only at runtime, point `DB\\\\\\\\\\\\\\\_PATH` and the`uploads/` folder at a persistent volume/disk (SQLite + local file uploadsneed writable, persistent storage — don't use ephemeral filesystems likedefault Vercel serverless functions).* Set all `.env` values as environment variables in your host's dashboardrather than uploading `.env`.* Put this behind HTTPS (most PaaS providers do this for you automatically).## Security notes* Passcode is bcrypt-hashed, never stored or compared in plaintext.* Admin routes require a valid JWT; tokens expire (`JWT\\\\\\\\\\\\\\\_EXPIRES\\\\\\\\\\\\\\\_IN`, default 12h).* Login endpoint is rate-limited (10 attempts / 15 min) to slow brute-forcing.* Global rate limit of 120 req/min per IP on the whole API.* CORS restricted to the origins you list in `CORS\\\\\\\\\\\\\\\_ORIGIN`.* Uploaded images are re-encoded via `sharp` (strips EXIF/metadata, resizes,re-compresses) rather than trusting the raw uploaded bytes.* `helmet` sets sane security headers by default.
